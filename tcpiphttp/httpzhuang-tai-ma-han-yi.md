| **状态码** | **含义** |
| :---: | :---: |
| **提供信息的** |  |
| 100 | 请求的开始部分已经收到，客户端可以继续它的请求 |
| 101 | 服务器同意切换协议 |
| **成功** |  |
| 200 | 请求成功 |
| 201 | 新的URL被创建 |
| 202 | 请求被接收，但还没有马上起作用 |
| 204 | 主体中没有内容 |
| **重新定向** |  |
| 301 | 服务器已经不再使用所请求的URL |
| 302 | 所请求的URL已暂时删除 |
| 304 | 文档还没有被修改 |
| **客户差错** |  |
| 400 | 在请求中有语法错误 |
| 401 | 请求缺少适当的授权 |
| 403 | 服务被拒绝 |
| 404 | 文档未找到 |
| 405 | 这个方法不被这个URL所支持 |
| 406 | 所请求的格式不可接受 |
| **服务器差错** |  |
| 500 | 在服务端有差错，比如奔溃 |
| 501 | 所请求的动作不能完成 |
| 503 | 服务暂时不可用 |

常见的状态码有：

* 200：响应成功
* 302：跳转，跳转地址通过响应头中的Location制定
* 400：客户端请求有语法错误，不能被服务器识别
* 403：服务器收到请求，但是拒绝提供服务（认证失败）
* 404：请求资源不存在
* 500：服务器内部错误
